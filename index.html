Investigación y aplicación de metodologías BEM y camelCase

Esta página tiene como objetivo explicar de forma concisa qué son, para qué sirven y cuándo usar BEM y camelCase, aplicando conocimientos básicos de HTML y CSS.

🧱 BEM 🧱

🧠 Explicación de qué es BEM 🧠

BEM (siglas de Block, Element, Modifier o en español, Bloque, Elemento, Modificador) es una metodología de nomenclatura para clases en CSS que busca mejorar la estructura, claridad y mantenibilidad del código.

🔢 Lista ordenada de las ventajas de usar BEM 🔢

  1. Facilita la lectura del código HTML y CSS gracias a su estructura clara y consistente.
  2. Evita colisiones de nombres entre estilos al mantener un contexto específico por bloque.
  3. Permite el desarrollo modular y reutilizable de componentes.
  4. Hace más sencillo el mantenimiento y escalado de proyectos grandes.
  5. Favorece el trabajo en equipo, ya que todos siguen la misma convención de nombres.

🔹 Lista desordenada de ejemplos de código usando BEM 🔹

  • boton – Bloque principal para un botón.
  • boton__icono – Elemento del bloque boton.
  • boton__icono--activo – Modificador que indica que el icono está activo.
  • menu__item – Elemento del bloque menu.
  • menu__item--seleccionado – Modificador que marca el ítem seleccionado.

🐪 camelCase 🐪

🧠 Explicación de qué es camelCase 🧠

camelCase es una convención de nomenclatura muy utilizada en programación, especialmente en JavaScript. Consiste en escribir frases o identificadores sin espacios, comenzando la primera palabra con minúscula y cada palabra siguiente con una letra mayúscula.

🔢 Lista ordenada de situaciones donde se usa camelCase 🔢

  1. let nombreUsuario = "Ana"; — Nombres de variables
  2. function calcularEdad() { } — Nombres de funciones
  3. persona.obtenerNombre(); — Métodos de objetos
  4. producto.precioTotal — Propiedades de objetos
  5. const listaDeCompras = []; — Identificadores en estructuras de datos

🔹 Lista desordenada de ejemplos de código usando camelCase 🔹

  • let nombreCompleto = "Carlos López";
  • const edadUsuario = 25;
  • function mostrarMensajePersonalizado() { }
  • const productosEnCarrito = ["camisa", "pantalón"];
  • usuario.obtenerDatos();

📌 ¿Cuándo usar cada metodología? 📌

Usa BEM cuando trabajes con HTML y CSS, especialmente si:

  • Estás creando interfaces visuales estructuradas.
  • Quieres evitar conflictos de estilos y mejorar la escalabilidad del código.
  • Trabajas en equipo y necesitas mantener una convención clara.

Usa camelCase cuando escribas JavaScript, especialmente para:

  1. Variables
  2. Funciones
  3. Objetos y métodos
